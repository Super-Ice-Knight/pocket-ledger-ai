# 答辩备忘

## 项目一句话

口袋记账 AI 版把中文消费描述解析成可确认的结构化账单，用 SQLite 保存真实流水，再由同一套主备模型分析本月预算和消费结构。

## 为什么前后端分离

React 负责页面、表单和状态反馈；FastAPI 负责数据校验、AI 调用、统计和数据库。API Key 只进入后端，前端看不到真实密钥。相比把所有代码放在一个脚本里，这种结构更容易说明每层责任。

## 为什么使用 SQLite

SQLite 是嵌入式关系数据库，数据保存在一个 `.db` 文件中，不需要额外启动数据库服务。单用户演示不需要 MySQL 或云数据库的运维成本，但仍然具备真实持久化、查询和约束。

本地数据库可长期保留。Render 免费实例没有持久磁盘时，重新部署或实例重建可能重置文件，因此线上定位为演示环境。

## 为什么金额保存整数分

浮点数不能可靠表示所有十进制小数。系统把 50 元保存为 5000 分，把 19.9 元保存为 1990 分：

```text
输入字符串 → Decimal/字符串拆分 → 四舍五入到整数分 → SQLite INTEGER
```

统计时只对整数求和，展示时再除以 100，因此不会出现 `0.1 + 0.2 = 0.30000000000000004`。

## AI 快记完整流程

1. 前端发送原始中文到 `POST /api/ai/parse-transaction`。
2. 后端先调用主 OpenAI 兼容接口。
3. 主模型超时、网络失败、JSON 错误或字段校验失败时调用备用模型。
4. 主备都失败后进入确定性本地规则。
5. 模型字段归一化到系统允许的分类和账户。
6. Pydantic 校验金额、类型、日期、长度和标签。
7. 前端显示来源、置信度和缺失字段。
8. 用户确认后才调用交易写入接口。

AI 不直接入账，因为模型可能误解金额、日期或账户。用户确认是数据正确性的最后边界。

## 三种 AI 状态

- `source=model, provider=primary|backup`：真实模型返回并通过校验。
- `source=local_rule, provider=local`：没有有效模型配置，主动使用本地规则。
- `source=error_fallback, provider=fallback`：主备模型都调用失败，临时回落本地规则。

界面按本次真实响应显示来源，不能只根据“配置了 Key”就声称正在使用模型。

## 本地金额规则如何避免误判

本地兜底不是取第一个数字，也不是随机语录。金额按优先级识别：

1. 明确单位或符号：`50元`、`50块`、`¥50`。
2. 金额语境：`花了50`、`收入2000`、`报销80`、`共50`。
3. 去掉日期、时间和计量数量后，只剩一个数字时才采用。
4. 仍有多个候选时返回金额缺失，不进行猜测。

例如：

```text
7月11日买了3杯咖啡，微信花了50元
```

系统识别 5000 分，而不是 700 分或 300 分。没有“花了50元”时，日期和杯数会被排除，金额进入 `missing_fields`。

## 分类、账户和备注怎么处理

- 分类使用关键词字典，例如咖啡、奶茶归为“饮品”，地铁、公交归为“交通”。
- 账户识别微信、支付宝、银行卡和现金。
- 收入关键词决定 `type=income`。
- 备注会删除金额、时间、账户和常见记账动词，保留消费场景。
- 模型返回 `books`、`alipay` 等别名时会映射为“学习”“支付宝”。
- 模型返回未知分类或账户时回落到本地结果，不允许任意值进入下拉框。

## 置信度是什么意思

本地规则识别出金额、分类和账户时置信度为 0.86；缺字段时为 0.52，并列出 `missing_fields`。模型可以返回自己的置信度，但无论多高都需要用户确认。

置信度不是服务可用性指标。低置信度不会触发备用模型；只有请求失败、响应异常或校验失败才会切换 Provider。

## AI 财务点评如何实现

`GET /api/ai/monthly-advice` 会把本月收入、支出、预算、分类、账户、趋势和最近流水作为统计 JSON 发给主备模型。角色 Prompt 要求：

- 面向学生和年轻用户。
- 只能依据提供的统计，不编造金额。
- 输出 `headline`、`detail` 和 `action_items`。
- `sharp` 可以直接但不能羞辱；`warm` 要温和、具体。

没有模型时，本地分析会根据预算是否设置、使用率、最高分类、主要账户、日均支出和剩余额度生成同结构结果。它是确定性分支，不是随机抽取语录。

## API Key 和设置页边界

本地开发：

- `RUNTIME_AI_SETTINGS_WRITABLE=true`。
- 设置页调用 `PUT /api/settings/ai`，配置保存在本地 SQLite。
- Key 输入框留空表示保留旧值。

公开演示：

- `RUNTIME_AI_SETTINGS_WRITABLE=false`。
- `PUT /api/settings/ai` 返回 403。
- Key 只保存在 Render 环境变量。
- 设置页只读展示模型、配置状态和连接测试。

`GET /api/settings/public` 和 `POST /api/settings/ai/test` 都不返回真实 Key。CORS 只能限制浏览器来源，不等于身份认证，因此无登录的公开演示必须关闭配置写入。

## API 挂了怎么办

模型 API 挂掉时，FastAPI 本身仍可用：

- 主模型失败后尝试备用模型。
- 主备都失败后返回本地规则草稿。
- 用户仍可手动填写并保存账单。

如果整个 Render 后端正在休眠，前端不会先显示虚假的零数据。超过约 1.2 秒会提示“后端正在唤醒”，网络失败后显示重新连接。GET 请求最多自动重试一次；POST、PUT、DELETE 不自动重试，避免重复入账。

## 非法输入会不会让系统崩溃

- 自然语言为空或超过长度时由 Pydantic 返回 422。
- 金额输入只允许两位小数，非法值在输入框旁提示。
- 金额、预算不能为负数。
- 类型只能为 `expense` 或 `income`。
- 标签去空、去重，最多 8 个，每个最多 16 字。
- 删除不存在的账单返回 404。
- 前端把 422、网络失败和超时转换为中文提示。

## Render 构建 Bug 怎么解决

Render 默认 Python 3.14 时，固定版本的 `pydantic-core` 没有匹配 wheel，pip 尝试通过 Rust/maturin 源码构建，并因只读缓存目录失败。项目通过 `.python-version` 和 `PYTHON_VERSION=3.11.9` 锁定已验证版本，而不是无目的升级依赖。

## 建议现场打开的代码

- `backend/app/money.py`：整数分与自然语言金额优先级。
- `backend/app/ai.py`：主备调用、字段归一化和本地兜底。
- `backend/app/runtime_settings.py`：环境变量与 SQLite 配置合并。
- `frontend/src/api.ts`：超时、GET 重试和中文错误。
- `frontend/src/App.tsx`：待确认草稿、来源标记和删除确认。
