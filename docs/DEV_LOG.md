- 将 OpenAI 兼容接口配置为 Agnes 的 `/v1` Base URL 和 `agnes-2.0-flash`，重启后端后 AI 快记返回 `source: model`。
- 修复模型把“今天”解析成旧日期的问题：Prompt 写入当前本地时间，同时后端对今天、昨天、前天做确定性日期覆盖。
- 增加相对日期单元测试，保证本地兜底和模型结果合并时不会破坏记账日期。
- 修复前端误连 8000 导致“后端 Key 未配置”的问题：新增 `frontend/.env.local` 指向 `http://127.0.0.1:8001`，并重启 Vite。
- Agnes 完整解析请求存在偶发慢响应，将后端 AI 请求超时从固定 15 秒改为 `AI_REQUEST_TIMEOUT_SECONDS` 配置项，本机设置为 45 秒。
- 接受备用模型设计：新增 SQLite `app_settings` 表保存真实运行配置，设置页通过 `PUT /api/settings/ai` 写入后端，不再只是 localStorage 演示。
- AI 调用改为主备 Provider 顺序：主模型异常后尝试备用模型，主备都失败才回落到本地规则；解析结果增加 `provider` 字段用于展示主模型、备用模型或兜底来源。
- 增加两条后端测试：设置保存不泄露 Key，主模型失败后会调用备用模型。
- 实测备用 API：发现 `deepseekv4-flash` 不是硅基流动有效模型 ID，通过 `/models` 修正为 `deepseek-ai/DeepSeek-V4-Flash`；主接口临时失效时成功返回 `provider: backup`。
- 针对备用模型可能返回英文或别名字段，新增后端归一化：`books/书籍 -> 学习`、`alipay -> 支付宝`、`微信支付 -> 微信`。
- 阶段评估后决定优先增强“可验证性”而非继续堆语音/App/OCR：新增 `POST /api/settings/ai/test`，设置页可一键测试主/备模型连接状态。
- 前端设置页新增主/备模型测试结果卡片，显示通断、模型名、耗时和简短错误，不展示 API Key。
- 后端测试增至 11 条，覆盖设置测试接口不泄露密钥。

## 2026-07-13

- 对线上 AI 延迟建立分层测量：预热后的 Render 健康接口约 170–184 ms，证明 FastAPI 与用户到后端链路正常；Provider 的 `latency_ms` 在服务端计时，可独立观察 Render 到模型平台的耗时。
- 记录 Agnes 在 45.2 秒后 `ReadTimeout`、硅基流动单次成功仍耗时 21.1 秒，以及真实快记曾等待 50.6–64.9 秒的现象；确认继续放宽超时不能解决体验问题。
- 将线上主模型迁移到 Groq。首次环境变量误填 `qwen3.6-27b`，接口在 221 ms 返回 `HTTP 404 model_not_found`；根据错误修正为带命名空间的 `qwen/qwen3.6-27b`。
- 最终线上配置为 Groq 单主模型、10 秒超时、公开设置只读、备用 Provider 关闭。代码仍保留可选备用接口能力，但不让已知慢接口进入关键链路。
- 最终公开环境复测：Groq 服务端测试 917 ms；普通中文快记 3.64 秒；“昨天兼职收入两千元，银行卡到账”4.87 秒并正确返回 200000 分、收入、银行卡和 `provider=primary`。
- 文档验收时发现月度点评连续进入 `error_fallback`。新增 Groq Qwen 非推理请求参数和包装 JSON 解码，补充两项回归测试；后端增至 20 项测试，部署后点评在 5.25 秒返回 `source=model, provider=primary` 与 3 条行动建议。
- 确认产品语义边界：一次快记只生成一笔账；同一句中的收入和支出不直接抵消，应分别入账后由统计计算净额。模型可以识别汉字金额，但本地兜底当前只保证阿拉伯数字。
- 点评从“页面加载自动调模型”改为“按需读缓存 + 手动生成”。新增 SQLite `ai_advice_cache`、数据指纹和 `fresh/stale/missing` 状态；数据改变后只标记过期，不自动消耗模型额度。
