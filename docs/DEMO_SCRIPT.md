# 演示脚本

## 目标

用 3 分钟讲清楚这个项目的核心价值：一句话 AI 记账、用户确认入账、SQLite 持久化、预算反馈、主备模型容灾和本地兜底。

## 讲解节奏

### 0:00 - 0:25 项目定位

这是一个面向学生日常消费的 AI 记账系统。它不是企业财务软件，而是解决“消费后懒得记、月底不知道钱花到哪里、AI 结果又不能直接相信”的问题。

技术上采用前后端分离：React 做交互界面，FastAPI 提供接口，SQLite 保存账单和配置，AI 使用 OpenAI 兼容接口。

### 0:25 - 1:05 一句话记账

切到“AI 快记”，输入：

```text
今天中午和室友吃疯狂星期四花了50块，微信付的，标签社交
```

点击解析后说明：

- 后端先调用主模型。
- 主模型失败才调用备用模型。
- 模型必须返回固定 JSON。
- AI 只生成草稿，不直接写数据库。
- 用户确认后才入账。

### 1:05 - 1:35 流水和标签

确认入账后切到“流水”。

说明流水按日期分组，每个组头展示当天笔数、支出、收入和净额。每笔账支持自定义标签，例如“社交”“小额高频”“宿舍”，这些标签是用户自己的分类补充，不依赖模型强行决定。

### 1:35 - 2:05 分析和预算

切到“分析”和“预算”。

说明图表旁边有文字结论，避免只展示一个图形。预算页会根据本月消费和预算上限判断风险，并展示温和或直接的建议。

### 2:05 - 2:40 设置和容灾

切到“设置”。

点击“测试主备接口”，说明：

- Key 不会回显。
- 设置保存在后端 SQLite。
- 主接口和备用接口都可以单独验证。
- API 不可用时，系统仍然保留本地规则解析和手动记账。

### 2:40 - 3:00 工程边界

最后强调：

- 金额用整数分保存，避免浮点误差。
- SQLite 适合单用户演示，部署时需要持久磁盘。
- 本地兜底基于金额正则、时间词、账户关键词和分类关键词，不是假装所有输入都由 AI 完成。
- `AI_LOG.md` 记录了提示词、采纳/拒绝理由、Debug 过程和验收证据。
